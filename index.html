First-Time Guide

start here
What this doesIt turns web research into sources, claims, blocked claims, and an answer a human can approve.
How to use itEnter a research question, build the packet, then read the claim ledger before using the answer.
What to trustTrust sourced claims more than the final paragraph. Refresh sources before final submission.
When to stopIf a claim is blocked, do not repeat it until a human approves updated evidence.

Review Path

source first
1. Pull sourcesStart from dated pages and source cards, not a free-floating answer.
2. Extract claimsTurn page evidence into a ledger with confidence and source IDs.
3. Block unsafe linesSeparate useful claims from claims that should not be repeated yet.
4. Call the humanExport an approval answer and handoff packet only after the risky lines are visible.
